1. Version Control System

Version Control System, disebut juga revision control system atau source code management adalah sistem yang mengelola perubahan dari sebuah dokumen program computer, website dan kumpulan informasi lain. Karena sistem ini merekam perubahan yang terjadi pada sebuah dokumen atau sekumpulan dokumen dari waktu ke waktu maka dimungkinkan untuk kembali ke versi tertentu di kemudian hari.

  • Local Version Control System

    • Metode version-control yang digunakan oleh banyak orang adalah menyalin dokumen ke folder lain. Pendekatan ini sangat umum karena sangat sederhana, tetapi juga sangat rentan terjadi kesalahan. Sangatlah mudah untuk lupa folder mana yang sedang digunakan dan secara tidak sengaja menulis ke dokumen yang salah atau menyalin dokumen yang tidak dimaksudkan. Untuk menyelesaikan masalah ini, programmer zaman dulu mengembangkan local VCSs yang memiliki sebuah database sederhana yang menyimpan seluruh perubahan terhadap dokumen dalam revision control.

  • Centralized Version Control System

    • Masalah besar lainnya yang orang-orang temui adalah mereka perlu berkolaborasi untuk mengembangkan sebuah sistem. Untuk menangani masalah ini, Centralized Version Control System dikembangkan. Sistem ini memiliki sebuah server tunggal yang berisi seluruh versi dari dokumen, dan sejumlah orang yang mengunduh dokumen dari tersebut. Sistem ini menawarkan berbagai kelebihan, khususnya jika dibandingkan lokal Version Control System. Misalnya, semua orang tahu hingga batas tertentu apa yang orang lain lakukan dalam proyek. Akan tetapi, sistem ini juga memiliki kekurangan yang besar. Jika server mati dalam beberapa jam, maka pada jam-jam tersebut orang-orang tidak bisa berkolaborasi ataupun menyimpan perubahan yang telah dikerjakan. Jika hard disk dari database sentral corrupt dan tidak ada cadangan yang disimpan, maka seluruh hal yang dikerjakan akan hilang, kecuali rekaman Tunggal yang dimiliki secara lokal.

  • Distributed Version Control System (DVCS)

    • Dalam DVCS (seperti Git, Mercurial, atau Darcs), klien tidak hanya mengunduh salinan terbaru dari dokumen; melainkan, menyalin seluruh repositori, termasuk riwayat lengkapnya. Oleh karena itu, jika server mati dan sistem-sistem ini berkolaborasi melalui server tersebut, repositori klien mana pun dapat disalin kembali ke server untuk memulihkannya. Setiap klon adalah cadangan lengkap dari semua data. Selain itu, banyak sistem ini mampu menangani beberapa repositori jarak jauh dengan baik, sehingga dapat berkolaborasi dengan kelompok orang yang berbeda secara bersamaan dalam proyek yang sama.

2. Git

Git adalah distributed version control software system yang mampu menangani perubahan dari source code atau data. Git sering digunakan untuk mengontrol source code oleh programmers yang secara berkolaborasi mengembangkan sistem. Terdapat tiga keadaan utama terkait keberadaan file: modified, staging, committed.

  • modified artinya berkas telah diubah namun belum men-commit-nya ke database
  • staging artinya file yang diubah pada versi saat ini telah ditandai untuk direkam pada commit selanjutnya
  • committed artinya data telah secara aman disimpan di lokal database

3. GitHub

Github adalah sebuah platform yang memungkinkan developer untuk membuat, menyimpan, mengatur serta membagikan kode yang dibuat. Github menggunakan git untuk menyediakan distributed version control dan github sendiri menyediakan kontrol akses, tracking bug, software feature request, task management, continues integration serta wikis untuk setiap proyeknya.
